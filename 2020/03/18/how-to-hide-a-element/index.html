how-to-hide-a-element

发表于 2020-03-18 分类于 WEB

隐藏一个元素是WEB开发中最普通最常见的需求了，通常我们可以简单的使用display: none来实现，这是最简单的通过CSS来完成，当然我们也可以通过Javascript移除DOM来模拟隐藏某一个元素。在这里我们只使用CSS。

这看似一个简单的问题，多数情况下并不会引起我们的注意，前端开发有点入门知识的初学者基本上都能通过display来实现，同样先前我也没有针对这个问题有过多的思考。

这里之所以重新来讨论这个问题，主要是因为在一个bug的调试中费了周折，迫使我不得不针对这个问题思考更多。该bug的情形是这样的，页面中存在一个<iframe>，当其src所指向的内嵌页面未完全加载完时，将<iframe>设置为display: none，待内嵌页面加载完，再将<iframe>设置为display: block，那么内嵌页面在渲染上会出现问题，似乎是内嵌页面不能按照预期的尺寸进行渲染。我同样也试了其他能内嵌页面的元素，比如<object>和<embed>，同样存在此问题。我的浏览器是Chrome 75.0.3770，这不能算作是古老的浏览器，不过我在Firefox中测试并未发现此问题。

显然问题就是由Chrome造成的。我们知道display: none的表现就像是该元素从未存在过一样，但它还是存在于源码中。我猜测Chrome是因为出于性能的考虑，对display: none的元素做了过多的限制，这仅仅是猜测，我并未考证。但是，我现在确确实实遇到了问题，而这个问题在Firefox中不存在，我越来越不喜欢Chrome。
